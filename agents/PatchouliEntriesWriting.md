# 帕秋莉手册单一条目撰写工作流程

## 工作目标
为 HexBoard 项目（一个Hexcasting附属）撰写一个帕秋莉手册条目，输出标准的 JSON 格式 entry 文件和对应的翻译文件。 

## 注意事项
- 条目是给玩家看的，不包含技术信息，仅于实际游玩使用相关。
- 如果涉及hexcasting本体内容，请向用户请求给予相关说明。


## 工作流程

### 第一步：确定条目主题
请告诉我您希望撰写的条目主题是什么？例如：
- 板块系统介绍
- 法杖使用方法
- 类型方块说明
- 配置选项指南
- 故障排除帮助
- 其他

等待用户回答后再进行下一步

### 第二步：定位相关文件
立即向用户询问该条目内容主要涉及项目的哪些文件？我需要了解：

1. **核心实现文件**
    - 相关的 Java 类文件路径
    - 主要的功能实现位置

2. **资源配置文件**
    - 物品定义文件
    - 合成配方文件
    - 语言文件
    - 模型文件

3. **其他相关内容**
    - 配置文件
    - 数据生成文件
    - 其他

记得询问用户！等待用户回答后可自行查找相关文件（包括但不限于用户给出的文件），然后进行下一步。

### 第三步：收集必要信息
我将从您指定的文件中提取：
- 物品 ID 和属性
- 合成配方
- 功能描述
- 使用方法
- 相关配置选项
- 其他相关内容

### 第四步：向用户确认相关内容  
我主要希望确认以下信息：
- 条目名称 `[entry_name]`
- 分类ID `[category]`
- 所需语言（可能多种）`[language]`
- 关键词的翻译
- 等重要信息
待用户确认后再进行接下来的工作

### 第五步：撰写条目内容

#### 5.1 创建 Entry JSON 文件
文件路径格式：`assets/hexcasting/patchouli_books/thehexbook/entries/[category]/[entry_name].json`

基本结构：
```json
{
  "name": "条目显示名称的翻译键",
  "category": "分类ID",
  "icon": "物品ID或图标路径",
  "pages": [
    {
      "type": "页面类型",
      "title": "页面标题（使用翻译键）",
      "text": "页面内容翻译键"
    }
  ]
}
```

注意事项：
- 对于类型为`patchouli:crafting`的页面，无需额外介绍合成表内容，也无需指定标题。


#### 5.2 创建翻译文件
文件路径：`assets/hexboard/lang/[language].json`

包含的翻译键：
- 条目名称
- 页面标题
- 页面内容文本
- 相关提示信息

所有的翻译键都使用如下命名规范：使用`page.hexboard.[entry_name].[index_number]`，其中的`[index_number]`是一个从0开始，按顺序增长的整数。

### 第六步：输出结果
我将为您提供：
1. 完整的 entry JSON 文件内容
2. 对应的语言文件翻译内容
3. 文件应该存放的具体路径
